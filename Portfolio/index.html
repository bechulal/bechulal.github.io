2D ACTION PLATFORMER

KILL THE HERO

UNREAL ENGINE 5  ·  BLUEPRINTS ONLY

// STORY

It started as a college project — a hero born to protect the world. But something went wrong. The original hero turned evil, consumed by darkness. A new champion rises with a single mission: hunt down the fallen legend and end the threat.

// GAMEPLAY

A fast-paced 2D action platformer where players move through levels and face a variety of enemies. The game focuses on fluid combat and platforming mechanics, all built using UE5's Blueprint system.

// HOW IT WAS MADE

My first Unreal Engine project — built entirely using Blueprint Visual Scripting with zero C++. Every mechanic from character movement and combat to enemy AI and level logic was wired up through UE5's node-based system. Enemy AI was designed using Behavior Trees and Blackboards, enabling structured and dynamic enemy decision-making. This project was a deep dive into Unreal's gameplay framework and laid the foundation for everything that came after.

3D ACTION

MONSTER HUNTER

UNREAL ENGINE 5  ·  C++ & BLUEPRINTS

// STORY

She doesn't farm, she doesn't trade — she hunts. The protagonist is a fearless monster hunter who survives entirely on the bounties she earns by taking down creatures across a dangerous world. No mercy. No rest. Just the hunt.

// GAMEPLAY

A 3D third-person action game featuring diverse monsters to track and fight. Players earn coins for each monster defeated. Multiple monster types are in the game, each with distinct behavior and challenge level.

// HOW IT WAS MADE

A significant step up from the first project, built primarily in C++ using Unreal Engine 5. Key systems built in the game:
  • Enemy AI state machine — enemies cycle through Patrol, Chase, Attack, Engaged, and Dead states, each with distinct logic and movement speeds
  • PawnSensing system — enemies detect the player using a configurable sight radius and peripheral vision angle, triggering dynamic pursuit
  • Directional hit reactions — dot product and cross product math calculates whether a hit came from front, back, left or right, playing the correct reaction animation
  • Motion Warping — attack animations smoothly snap the character into position relative to the combat target using warp translation and rotation targets
  • Enhanced Input System — player input is handled via UE5's modern Enhanced Input system with full action mapping support
  • Action state machine — a state system on the player prevents input conflicts between attacking, dodging, equipping, and hit reactions
  • Stamina system — stamina regenerates per tick and is consumed by dodge rolls, with real-time HUD feedback
  • Weapon socket system — weapons attach to hand or back sockets via animation-driven equip/unequip montages
  • Component-based architecture — health, stamina, gold, and souls are managed through a custom AttributeComponent shared across all characters
